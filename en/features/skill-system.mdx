## Overview

Decepticon agents don't operate from hardcoded playbooks. They draw from a **skill system** — a curated library of offensive techniques organized by kill chain phase and tagged with MITRE ATT&CK identifiers.

## Progressive Disclosure

Skills use a **progressive loading** architecture to keep agent context windows clean:

1. **Initial load** — Only skill frontmatter (name, description, ATT&CK tags, phase) is loaded
2. **On-demand** — When an agent selects a skill, the full content is loaded via `read_file()`
3. **Clean context** — Agents only carry the detail they need for their current objective

<Info>
  This prevents context bloat. An agent working on initial access doesn't need the full content of lateral movement techniques — just their existence and metadata.
</Info>

## Kill Chain Coverage

Skills are organized by offensive phase:

| Phase | Examples | ATT&CK Tactics |
|-------|----------|-----------------|
| **OSINT** | Domain enumeration, social engineering recon | Reconnaissance (TA0043) |
| **Web Exploitation** | SQL injection, authentication bypass, SSRF | Initial Access (TA0001) |
| **Active Directory** | Kerberoasting, AS-REP roasting, DCSync | Credential Access (TA0006) |
| **Privilege Escalation** | Kernel exploits, SUID abuse, token manipulation | Privilege Escalation (TA0004) |
| **Lateral Movement** | Pass-the-hash, WMI exec, PSRemoting | Lateral Movement (TA0008) |
| **Credential Access** | Mimikatz, LSASS dump, credential files | Credential Access (TA0006) |
| **Defense Evasion** | Process injection, timestomping, log clearing | Defense Evasion (TA0005) |
| **OPSEC** | Traffic patterns, artifact cleanup, stealth validation | Defense Evasion (TA0005) |

## Skill Frontmatter Example

```yaml
name: kerberoasting
description: Extract and crack service account TGS tickets
phase: credential-access
mitre:
  - T1558.003  # Kerberoasting
tools:
  - impacket-GetUserSPNs
  - hashcat
prerequisites:
  - Valid domain user credentials
  - Network access to Domain Controller
```

The agent sees this metadata when browsing available skills. Only when it selects Kerberoasting does the full technique content — commands, detection notes, variations — get loaded into context.
